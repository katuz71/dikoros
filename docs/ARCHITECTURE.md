## Ключевые решения (резюме ADR)

### ADR №001: Уход от Хорошопа
Хорошоп ограничивает мобильное приложение и кастомизацию. Строим собственное решение на Medusa + Next.js.

### ADR №002: Next.js 15 + App Router
Выбран Next.js 15 (актуальная стабильная версия на момент старта). App Router — для server components и нативной поддержки в Payload 3.

### ADR №003: Payload CMS встроен в Next.js
Payload 3 поддерживает встроенный режим — один процесс Next.js обслуживает и фронт, и админку CMS. Минус один сервис в Docker, проще деплой и SSR.

### ADR №004: PostgreSQL — одна инстанция, две схемы
Medusa и Payload делят один Postgres, но разные схемы (`medusa`, `payload`). Снижает расходы и упрощает бэкап.

### ADR №005: Hetzner CX22 на старте
Дёшево (~€4/мес), достаточно для staging и первого продакшена. Вертикальный апгрейд при необходимости.

### ADR №006: Docker Compose, не Kubernetes
Один человек в команде → K8s избыточен. Compose покрывает все нужды до ощутимого роста.

### ADR №007: Caddy как reverse proxy
Автоматический Let's Encrypt, простая конфигурация, не нужен отдельный certbot.

### ADR №008: Cloudflare R2 для медиа и бэкапов
S3-совместимое API, нет egress-трафика, дешевле AWS S3.

### ADR №009: Resend для email
Простой API, хорошие лимиты на бесплатном плане, отличная DX.

### ADR №010: Sentry + UptimeRobot для мониторинга
Free-планы покрывают потребности на старте. Альтернативы (Grafana/Prometheus) избыточны для одного человека.

### ADR №011: CRM (OneBox) — под пересмотром
Решение после аудита текущей системы. Возможные альтернативы: KeyCRM, остаться на OneBox с переписанной интеграцией, минимальный admin в Medusa.

### ADR №012: Чат-бот — отложен на Месяц 3
В фокусе Месяцев 1–2 — миграция и инфраструктура.
