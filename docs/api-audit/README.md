# API Audit — DIKOROS

Аудит текущего API проекта DIKOROS (FastAPI на `app.dikoros.ua`), выполнен в Неделе 1 (Этапы 2–3) перед миграцией на Medusa.js.

Дата аудита: 5 мая 2026.

## Что в этой папке

### Источники данных

- **`openapi.json`** — снят с `https://app.dikoros.ua/openapi.json` через curl. OpenAPI 3.1.0, 53 КБ. Источник истины для всех 89 операций бэкенда.
- **`openapi-flat.txt`** — плоский список всех путей и методов из openapi (по одной строке на операцию). Удобно для `Select-String`.
- **`admin-calls-raw.txt`** — все 32 строки с `fetch(...)` из `admin.html`, как они есть в коде.
- **`admin-paths-raw.txt`** — литеральные пути из admin.html (12 строк, без шаблонов).
- **`contract-raw.md`** — сырая Markdown-таблица контракта на 90 операций + 27 схем, без разметки. Артефакт Этапа 2, оставлен для истории.

### Результаты анализа

- **`contract.md`** — главный документ. Финальная таблица всех операций с разметкой `USED / ADMIN / SERVICE / DEAD`, разнесённая по 14 доменам. Колонки: Method, Path, Status, Summary, Notes. Сюда смотрим первым делом.
- **`findings.md`** — 12 архитектурных наблюдений и рекомендаций. Безопасность, дубли путей, отсутствие схемы Product, push-уведомления, ключ Новой Почты, ручка `/delete-account` как блокер для сторов. Каждый пункт в формате «что нашли → почему важно → что делать при миграции».
- **`by-domain/`** — нарезка контракта по аудиториям:
  - `storefront.md` — все USED-операции (контракт мобильного приложения, 17 ручек).
  - `admin.md` — все ADMIN-операции (контракт текущей admin.html, 34 ручки).
  - `infrastructure.md` — SERVICE и DEAD (health, webhooks, статика, мёртвые дубли, заброшенные фичи).

## Сводка цифр

- Всего операций в openapi: **89**
- Плюс «невидимых» в openapi (есть в admin.html, нет в schema): **2** (`POST /api/import_xml`, `POST /categories/{id}/banner`)
- Уникальных вызовов из RN-приложения: **17**
- Уникальных вызовов из admin.html: **24** пути / 32 операции

Распределение по статусам:
- USED: 17 (мобайл)
- ADMIN: 34 (веб-админка)
- SERVICE: 7 (health, webhooks, статика)
- DEAD: 31 (дубли и заброшенное)

## Что было сделано

1. Снят `openapi.json` с публичного эндпоинта прода.
2. Скачан snapshot прод-кода с сервера (10 МБ, в `~/dikoros-audit/source-snapshot/`, в репо не коммитится).
3. Грепом по 8 папкам RN-фронта найдены 50 совпадений с `fetch(`/`axios.`/`API_URL`, нормализованы в 17 уникальных API-вызовов.
4. Грепом по `admin.html` найдены 32 операции, нормализованы в 24 уникальных пути.
5. Сведены три источника (openapi, RN, admin) в единую таблицу `contract.md` с разметкой статусов.
6. Записаны архитектурные наблюдения в `findings.md`.
7. Сделана нарезка по аудиториям в `by-domain/`.

## Что НЕ было сделано (намеренно)

- **Не открывался `main.py`** (260 КБ, ~5000 строк) — анализ через `openapi.json` дал полную картину контракта без чтения кода. К main.py возвращаемся только для разрешения двух «невидимых» ручек и проверки опасных эндпоинтов из findings.
- **Не извлекались тела запросов и ответов в детальных схемах** — для миграции мобайла будут сняты с прода через curl как fixtures на этапе разработки API-слоя Medusa (Месяц 2).
- **Не проводилось функциональное тестирование** — это аудит контракта, не QA.
