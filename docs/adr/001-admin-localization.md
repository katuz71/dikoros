# ADR 001 — Локализация Medusa Admin (UA/RU) через JSON i18n

- **Дата:** 9 мая 2026
- **Статус:** Принято
- **Контекст:** этап 10 закрыт (Medusa 2.14.2 поднята), клиент требует админку на украинском.

## Контекст

Клиент (Юра Велигура) не принимает английскую админку. Нужен украинский интерфейс Medusa Admin. На этапе bootstrap всплыло предложение мигрировать на Saleor из-за «лучшей локализации».

Проверка показала: Saleor Dashboard использует Transifex, украинский локаль неполный (issue #6425), русского нет; для собственной локализации требуется форк дашборда с merge-конфликтами на каждом релизе.

Medusa Admin с версии 2.11.1 поддерживает официальный механизм i18n на базе `react-i18next` (https://docs.medusajs.com/learn/fundamentals/admin/translations). У нас установлена 2.14.2 — механизм доступен.

## Решение

Остаёмся на Medusa. Локализуем Admin UI через JSON-файлы в монорепе:


apps/backend/src/admin/i18n/ ├── index.ts ├── i18next.d.ts └── json/ ├── en.json └── uk.json

Подключение через `useTranslation()` в виджетах и UI-routes. Файлы лежат в нашем коде и не перезаписываются `pnpm update @medusajs/*`.

## Альтернативы

1. **Миграция на Saleor.** Отклонено: ~30 ч работы выкинуть, сменить TS-моноrepo на Python, локаль всё равно неполная, форк дашборда повышает стоимость каждого релиза в 2–8 ч.
2. **Оставить английскую админку.** Отклонено клиентом.
3. **Контрибьютить украинский в upstream Medusa.** Отложено на потом (20+ ч, требует процедуры PR + ревью). После запуска можно вынести наш `uk.json` в апстрим.

## Последствия

**Плюсы:**
- Стек не меняется, ~30 ч работы сохранены.
- Файл переводов в репозитории — полный контроль, версионирование, ревью через PR.
- Минорные апгрейды Medusa не ломают переводы (отсутствующие ключи фолбэчатся в EN).
- Можно добавить `ru.json` за ~30 минут, если потребуется.

**Минусы / риски:**
- Каждый минор Medusa приносит ~20–100 новых строк UI (1–3% объёма) → ~15–30 минут перевода раз в 1–2 месяца.
- Редкие переименования ключей в апстриме → фолбэк показывает имя ключа; чинится grep’ом.
- Первоначальный объём 1500–2000 строк: ~1 ч машинного перевода (DeepL/GPT) + 4–6 ч ручной вычитки ключевых экранов (заказы, товары, клиенты, настройки).
- Украинская плюрализация (`_one`, `_few`, `_many`, `_other`) для ~30–50 строк → ~2 ч ручной работы.

**Итого первичная оценка:** 8–12 ч (задача `w4-admin-ua-translation`), ~30 мин/мес поддержка, финальная вычитка перед prod в месяце 5 (`m5-admin-ua-translation-polish`, 4 ч).

## Связанные задачи (Notion)

- `w4-admin-ua-translation` — Месяц 1, Неделя 4 (10 ч).
- `m5-admin-ua-translation-polish` — Месяц 5 (4 ч).
